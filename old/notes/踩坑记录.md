# LangChain 学习踩坑记录

## Day 1 - Models + PromptTemplate

### 问题 1：MiniMax API 兼容性问题
**现象**：ChatOpenAI 调用报错
**原因**：MiniMax 的 API 格式与标准 OpenAI 有差异
**解决**：检查 base_url 和 model 参数是否正确

### 问题 2：Message 类型混用
**现象**：模型回答混乱
**原因**：HumanMessage / AIMessage / SystemMessage 混用
**解决**：明确区分消息类型，SystemMessage 放最前面

---

## Day 2 - Chains（LCEL）

### 问题 1：管道操作符类型不匹配
**现象**：`|` 管道连接后调用失败
**原因**：中间步骤输出格式与下一步输入不匹配
**解决**：使用 `| StrOutputParser()` 转换格式

### 问题 2：stream 和 invoke 区别
**现象**：不知道用哪个
**解决**：
- `invoke()`：一次性返回完整结果
- `stream()`：流式输出，适合长文本

---

## Day 3 - Memory

### 问题 1：Memory 不自动传入 Prompt
**现象**：模型不知道上文
**原因**：需要在 Chain 初始化时指定 `memory` 参数
**解决**：`ConversationChain(llm=llm, memory=memory)`

### 问题 2：Token 超出限制
**现象**：对话历史太长后报错
**解决**：
- 使用 `ConversationSummaryMemory` 摘要压缩
- 使用 `ConversationTokenBufferMemory` 按 token 切

---

## Day 4 - RAG

### 问题 1：Embedding 服务不可用
**现象**：FAISS 建库失败
**原因**：MiniMax 可能不支持 Embedding API
**解决**：使用 OpenAI Embedding 或其他支持的服务

### 问题 2：相似度检索结果不准确
**解决**：
- 调整 `chunk_size` 和 `chunk_overlap`
- 优化 Prompt 设计
- 尝试不同的 `k` 值

---

## Day 5 - Agents

### 问题 1：Agent 进入死循环
**现象**：Agent 不断调用同一个 Tool
**原因**：Tool 返回结果后 Agent 不知道何时停止
**解决**：在 Prompt 中明确终止条件

### 问题 2：Tool 参数定义错误
**现象**：Tool 调用失败
**原因**：LLM 生成的参数格式不对
**解决**：使用 Pydantic BaseModel 正确定义 Tool Schema

---

## Day 6 - 完整项目

### 问题 1：多轮对话指代词解析
**现象**："刚才那个指标"模型不认识
**解决**：
- 依赖 Memory 自动注入历史
- 在 Prompt 中显式传递历史上下文

### 问题 2：SQL 执行安全
**现象**：担心生成恶意 SQL
**解决**：
- 不直接执行 LLM 生成的 SQL
- 用 LLM 生成意图，用确定性规则生成 SQL

---

## 通用经验

1. **Always define output schema**：使用 Pydantic 或 JSON Schema 约束输出
2. **Start simple**：先跑通最简单版本，再逐步加复杂度
3. **Use Memory wisely**：不是所有场景都需要 Memory，按需使用
4. **Test edge cases**：测试指代词、上下文丢失等边界情况
5. **Monitor token usage**：注意 API 调用成本
